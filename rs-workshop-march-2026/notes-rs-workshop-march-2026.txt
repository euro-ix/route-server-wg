# Claudio on OpenBGPD performance 
#

- In a more detailed presentation Claudio went through the performance points of OpenBGPD

- Reduced size of data structures to better fit into cache lines

- Reduced number of cache misses by rearranging the data structures by using a linear array instead of a linked list

- more than 50% speedup

- better memory consumption by half 

- dropped latency from minutes to seconds

- Next steps for filters 

    - outbound filters are improved but not inbound 

    - they are dependent heavily on communities

    - Rework the Adj-RIB-In & Loc-RIB to benefit in a similar way to the Adj-RIB-Out

    - Reconsider pending updates & withdraw queues

    - Better to store updates for groups of peers & not use per-peer queues
